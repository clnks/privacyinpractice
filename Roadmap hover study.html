Accessibility study · roadmap.html

The hover state is making your roadmap unreadable for some users.

On cream backgrounds, the lavender accent measures roughly 1.4 : 1 against the page — well below the 4.5 : 1 WCAG AA minimum for body text and 3 : 1 for large text. On hover, the phase title shifts toward that lavender, so the action actively reduces legibility. Below: five replacements, ordered from minimal to bold.

Where it fails

.phase:hover .phase-title → color: var(--accent) (#c4b1dd on #f1e8d7). Also .phase-tag, li::before numbered chips, .deliverable left border + label.

Where it’s fine

Lavender on --plum (dark block) measures ~10 : 1 — that section is good. So the fix is not to abandon lavender; it’s to swap it for a deeper sibling on light backgrounds.

What I'd actually change in your CSS

  • Add a deeper sibling token: --accent-strong: #4a2d7a; in :root. Keep --accent (lavender) for use on dark backgrounds and as a fill tint.
  • On cream sections, replace color: var(--accent) with color: var(--accent-strong) for: .phase-tag, .phase-body li::before, .deliverable b, .deliverable left border, .cert .status, .section-head .num.
  • Replace .phase:hover .phase-title { color: var(--accent) } with one of the patterns above. My pick: variant 03 (ink bar + paper fill) for the cream tracks, and the existing lavender hover stays on the plum track (it already works there).
  • Add .phase:focus-visible mirroring whatever hover does — keyboard parity.
  • Wrap transitions in @media (prefers-reduced-motion: no-preference) so v6-style motion is opt-out.
  • Optional: make the whole .phase a <button> or anchor so it’s reachable in the tab order — right now nothing on the roadmap is keyboard-focusable.
